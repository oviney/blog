# Architecture

A high-level overview of the systems that make up https://www.viney.ca/.

## System Diagram

```
┌─────────────────────────────────────────────────────────────┐
│  Content & Code (GitHub: oviney/blog)                        │
│                                                              │
│  _posts/ ──┐                                                 │
│  _layouts/ ─┤                                                │
│  _sass/    ─┼─► Jekyll 4.3.2 build ──► GitHub Pages         │
│  _includes/ ─┤                          (www.viney.ca)       │
│  _config.yml ┘                                               │
│                                                              │
│  GitHub Actions                                              │
│  ├── jekyll.yml        (deploy on push to main)              │
│  ├── test-build.yml    (build + HTML validation)             │
│  ├── test-quality.yml  (visual, a11y, Lighthouse, security)  │
│  └── healing-monitor.yml (Playwright healing, every 4h)      │
└─────────────────────────────────────────────────────────────┘
```

## Components

### Jekyll (Static Site Generator)

- **Version**: 4.3.2
- **Config**: `_config.yml`
- **Build**: Runs in GitHub Actions on every push to `main`
- **Output**: Static HTML/CSS/JS deployed to GitHub Pages

**Key directories**:

| Path | Contents |
|---|---|
| `_posts/` | Blog articles (Markdown with YAML front matter) |
| `_layouts/` | HTML templates (`default.html`, `post.html`) |
| `_sass/` | SCSS stylesheets (`economist-theme.scss`) |
| `_includes/` | Reusable HTML fragments |
| `_data/` | Structured data (YAML/JSON) |
| `assets/` | Static files (images, fonts, JS) |

### Economist Theme

A custom design system (`_sass/economist-theme.scss`, 600+ lines) inspired by The Economist's visual identity.

- **Typography**: Merriweather (serif) for body, Inter (sans-serif) for UI — loaded from Google Fonts
- **Colours**: Economist red `#E3120B` for accents, `#1a1a1a` text on white
- **Layouts**: `default.html` wraps all pages; `post.html` extends it for articles
- **Responsive**: Mobile-first SCSS with breakpoints at 768px and 1024px

See [`docs/skills/economist-theme/SKILL.md`](docs/skills/economist-theme/SKILL.md) for design guidelines.

### GitHub Actions

Four workflows under `.github/workflows/`:

| Workflow | Trigger | Purpose |
|---|---|---|
| `jekyll.yml` | Push to `main` | Build and deploy to GitHub Pages |
| `test-build.yml` | Push / PR | Jekyll build validation + HTML checks |
| `test-quality.yml` | Push / PR | BackstopJS visual regression, pa11y-ci, Lighthouse CI, npm audit |
| `healing-monitor.yml` | Schedule (every 4 h) + push | Run Playwright healing tests; update dashboard data |

### Playwright Testing & Healing

End-to-end tests live in `tests/` and `specs/`. The healing system automatically repairs fragile selectors when tests fail.

- **Config**: `playwright.config.ts`
- **Specs**: `tests/`, `specs/`
- **Healing scripts**: `scripts/`
- **Dashboard**: `dashboard/` — static HTML page at `/dashboard/` showing healing metrics

### Monitoring Dashboard

A static HTML dashboard (`dashboard/index.html`) visualising Playwright healing success rates and trends.

- **Data**: `dashboard/dashboard-data.json` — regenerated by `healing-monitor.yml` every 4 hours
- **Charts**: Chart.js
- **Access**: https://oviney.github.io/blog/dashboard/
- **Page refresh**: every 5 minutes (client-side); underlying data refreshes every 4 hours

### Content Generation

Articles are drafted by [economist-agents](https://github.com/oviney/economist-agents), a separate multi-agent AI system that generates content in The Economist's editorial voice. All AI-generated content is reviewed and approved by a human editor before publication. An AI disclosure appears on assisted articles.

## Quality Gates

Changes to `main` must pass all automated checks before serving live traffic:

1. **Pre-commit** (local): Jekyll build validation
2. **test-build.yml**: Jekyll build + HTML validation
3. **test-quality.yml**: Visual regression (BackstopJS), accessibility (pa11y-ci WCAG 2.1 AA), performance/SEO (Lighthouse CI), security (npm audit)
4. **Deployment**: Jekyll build in `jekyll.yml`; if it fails, the previous version stays live
