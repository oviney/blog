# SEO Feedback Loop — Google Search Console

## Overview

This document describes how to verify **www.viney.ca** in Google Search Console (GSC),
submit the sitemap, and run the monthly traffic review that feeds back into content decisions.

---

## 1. Verify the Site in Google Search Console

The blog supports HTML-tag verification via a meta tag injected in `<head>`.

### Steps

1. Go to [Google Search Console](https://search.google.com/search-console/) and sign in.
2. Click **Add property** and enter `https://www.viney.ca/`.
3. Choose the **HTML tag** verification method.
4. Copy only the `content` value from the provided `<meta>` tag (e.g. `abc123XYZ…`).
5. Open `_data/analytics.yml` in the repository and set:
   ```yaml
   google_search_console_verification: "abc123XYZ…"
   ```
6. Commit the change and wait for GitHub Actions to deploy the site.
7. Return to Search Console and click **Verify**.

> The meta tag is rendered by `_includes/analytics.html` — it is omitted entirely when
> the value is blank, so removing the value later will cleanly remove the tag.

---

## 2. Submit the Sitemap

Once verification succeeds:

1. In Search Console, navigate to **Sitemaps** (left sidebar).
2. Enter the sitemap URL: `sitemap.xml`
3. Click **Submit**.

Jekyll auto-generates `/sitemap.xml` via the `jekyll-sitemap` gem listed in `_config.yml`.
The full URL is `https://www.viney.ca/sitemap.xml`.

## 3. Monthly Traffic Review

Run this review on the **first Monday of each month**.
Open a new issue using the [Monthly Traffic Review template][template] to track findings.

[template]: https://github.com/oviney/blog/issues/new?template=monthly-traffic-review.yml

### Metrics to check

| Source | Metric | Where to find |
|--------|--------|---------------|
| Google Search Console | Organic impressions | Performance → Search results |
| Google Search Console | Click-through rate (CTR) | Performance → Search results |
| Google Search Console | Top queries | Performance → Queries tab |
| Google Search Console | Index coverage errors | Index → Pages |
| Google Analytics (GA4) | Sessions from organic search | Reports → Acquisition → Traffic acquisition |
| Google Analytics (GA4) | Avg. engagement time | Reports → Engagement → Pages and screens |
| Google Analytics (GA4) | Top landing pages | Reports → Engagement → Pages and screens |

### Interpreting the data

- **High impressions, low CTR** → Improve title tags and meta descriptions for those posts.
- **High CTR, low engagement time** → The content isn't matching search intent; revise the post.
- **Top queries not matching any post** → Write new content targeting those queries.
- **Index coverage errors** → Fix broken pages or add missing canonical tags.

---

## 4. Feeding Data Back into Content Decisions

After each monthly review, update the content backlog:

1. Pull the **top 10 queries** from Search Console for which the site ranks on page 2 or 3.
2. For each query, decide:
   - **Existing post** that can be updated with more depth → Open an `agent:editorial-chief` issue.
   - **No matching post** → Create a new post proposal issue with the `agent:editorial-chief` label.
3. Prioritise posts that intersect high-volume queries with the site's four categories:
   `Quality Engineering`, `Software Engineering`, `Test Automation`, `Security`.
