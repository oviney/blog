# decisions.md — Architectural Decision Log

This file records architectural decisions that span agent sessions.
Agents should consult this file before making changes that touch site-wide
configuration, design conventions, or content standards.

---

## ADR-001: Economist-Inspired Design Theme

**Date**: January 2026
**Status**: Active
**Context**: The blog needed a distinctive, professional visual identity.
**Decision**: Implement a custom Sass theme (`_sass/economist-theme.scss`) modelled on The Economist's visual identity — Economist red (`#E3120B`) as accent, Merriweather serif for body text, Inter sans-serif for UI elements, and a single-column layout capped at 1040 px.
**Consequences**: All agents must use SCSS variables from `_sass/economist-theme.scss` and never hardcode colours, fonts, or spacing values.

---

## ADR-002: Normalised Category Taxonomy

**Date**: January 2026
**Status**: Active
**Context**: Early posts used inconsistent category strings, breaking topic-page filters and search.
**Decision**: Restrict categories to exactly four values: `Quality Engineering`, `Software Engineering`, `Test Automation`, `Security`. No other values are permitted.
**Consequences**: The Editorial Chief must validate categories in front-matter before publishing. New categories require a deliberate decision (update this file, update topic pages, and notify the Creative Director).

---

## ADR-003: Jekyll 4 + GitHub Actions (not GitHub Pages auto-build)

**Date**: December 31, 2025
**Status**: Active
**Context**: The blog started on Jekyll 3 / GitHub Pages automatic build, which constrained plugin selection and deployment control.
**Decision**: Migrate to Jekyll 4.3.2 built and deployed via a custom GitHub Actions workflow (`.github/workflows/`). GitHub Pages is configured to serve the output of that workflow, not from a branch.
**Consequences**: `bundle exec jekyll build` is the canonical validation step. The QA Gatekeeper owns the workflow files. No agent may revert to a branch-based Pages deployment without a new ADR.

## ADR-004: Multi-Agent System with Shared Memory Files

**Date**: April 2026
**Status**: Active
**Context**: Multiple AI agents (Creative Director, QA Gatekeeper, Editorial Chief, General) operate across independent sessions with no shared runtime state. Without coordination, agents make conflicting or redundant changes.
**Decision**: Adopt the Squad pattern for cross-session coordination: `AGENTS.md` documents each agent's role and boundaries; `decisions.md` (this file) logs architectural decisions. Both files are stored at the repo root so every agent reads them before working.
**Consequences**: All agents must update `decisions.md` when a new architectural decision is made. Agents must read both files at the start of each session.

## ADR-005: AI-Assisted Content with Human Editorial Approval

**Date**: January 2026
**Status**: Active
**Context**: Blog posts are partly generated by the Editorial Chief AI agent, raising questions about transparency and accuracy.
**Decision**: All AI-generated or AI-assisted content is disclosed via the `ai_disclosure` field in `_config.yml`. Every post still requires human review, fact-checking, and editorial approval before publication. The disclosure text is rendered in the site footer.
**Consequences**: The Editorial Chief must not publish posts directly to `main` without human sign-off. Draft posts live in `_drafts/` until approved.

---

## ADR-006: No Remote Theme in Production Build

**Date**: January 2026
**Status**: Active
**Context**: The site initially used `remote_theme: mmistakes/minimal-mistakes@4.27.3`, which added network dependency and layout constraints incompatible with the Economist design.
**Decision**: Commented out `remote_theme` in `_config.yml` and replaced Minimal Mistakes layouts with custom Economist-style layouts in `_layouts/`. The design system is fully self-contained in `_sass/economist-theme.scss`.
**Consequences**: The Creative Director owns all layout and SCSS files. Adding a remote theme again requires a new ADR and human approval of `_config.yml`.

---

## ADR-007: Pre-commit Hook for Jekyll Build Validation

**Date**: December 2025
**Status**: Active
**Context**: Broken builds were occasionally pushed to the main branch, blocking deployment.
**Decision**: A pre-commit hook runs `bundle exec jekyll build` before every commit. The hook is defined in the repository and must remain in place.
**Consequences**: Agents must ensure `bundle exec jekyll build` succeeds locally before committing. The QA Gatekeeper is responsible for maintaining the hook configuration. Disabling the hook requires explicit human approval.
